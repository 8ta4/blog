First, `println` sometimes gets swallowed by your ClojureScript REPL. I work around this issue by configuring Neovim's Node client to write to a log file [using the `NVIM_NODE_LOG_FILE` environment variable](https://github.com/8ta4/word/blob/57e47bff5404d6d2c81cc4a6466b901303a9b764/devenv.nix#L49). I [tail this file](https://github.com/8ta4/word/blob/57e47bff5404d6d2c81cc4a6466b901303a9b764/devenv.nix#L39) in a split terminal. I'd have gone insane without a REPL. Thankfuly, REPL a day keeps the doctor away.

Second, a Neovim client instance [gets passed](https://github.com/8ta4/word/blob/57e47bff5404d6d2c81cc4a6466b901303a9b764/src/main.cljs#L591) to your entry point function. I [store](https://github.com/8ta4/word/blob/57e47bff5404d6d2c81cc4a6466b901303a9b764/src/main.cljs#L598-L606) the instance inside an atom during initialization. This reference lets me evaluate forms in the REPL with a live editor.

Third, it's tedious to run `:UpdateRemotePlugins` by hand. I set up [a hook](https://github.com/8ta4/word/blob/57e47bff5404d6d2c81cc4a6466b901303a9b764/shadow-cljs.edn#L12) in `shadow-cljs.edn`. After compiling, `shadow-cljs` [invokes](https://github.com/8ta4/word/blob/57e47bff5404d6d2c81cc4a6466b901303a9b764/src/build.clj#L26) Neovim in headless mode to update the remote plugin registration.

I set up [this repo](https://github.com/8ta4/word) using the tricks I talked about. Do you have any other tips? Feel free to share your thoughts!
